## 7. Dependencias ML.NET en WebForms

Se tuvo que agregar `Microsoft.ML` al proyecto WebForms.

También apareció el error

```text
The type 'Attribute' is defined in an assembly that is not referenced.
You must add a reference to assembly 'netstandard'
```

Corrección aplicadarecomendada en `Web.config` dentro de `compilation`

```xml
assemblies
  add assembly=netstandard, Version=2.0.0.0, Culture=neutral, PublicKeyToken=cc7b13ffcd2ddd51 
assemblies
```

También apareció el error

```text
No se puede cargar el archivo DLL 'CpuMathNative'
```

Corrección aplicada

copiar

```text
CpuMathNative.dll
```

desde

```text
packagesMicrosoft.ML.CpuMath.4.0.2runtimeswin-x64native
```

hacia

```text
Intranet.WorkflowStudio.WebFormsbin
```

Si IIS Express corriera en 32 bits, habría que usar `win-x86`, pero la corrección con `win-x64` permitió avanzar.

## 9. Fixes realizados durante esta etapa

### fix3 — Resolver runtime

Objetivo

Resolver palabras del usuario contra datos reales del sistema.

Validado

```text
NC → NOTA_CREDITO_ELECTRONICA_AR
compras → COMPRAS
dirección → DIR_GENERAL
administración → ADM_FIN
operaciones → OPERACIONES
it  sistemas  informática → IT
```

También se agregó soporte para resolver usuarios desde `WF_User` cuando el texto coincida con `UserKey` o `DisplayName`.

### fix4 — Corrección de importe

Problema

```text
si supera 200000
```

se interpretaba como

```text
200
```

Corrección

Ahora interpreta correctamente

```text
200000
```

Formatos contemplados

```text
200000
200.000
200,000
$200000
$ 200.000
200000,50
200.000,50
```

Validado

```text
Total mayor a 200000
```

### fix5 — Branch Planner

Objetivo

Detectar ramas lógicas en frases complejas.

Ejemplo interpretado

```text
si no tiene CAE → Compras
si supera 200000 → Dirección
si no supera 200000 → Administración
```

Se agregó en JSON técnico

```json
branchPlan {
  planner runtime-branch-planner-v1,
  hasBranches true,
  branches []
}
```

También se agregó visualmente la sección

```text
Plan de ramas
```

### fix6 — Scroll panel Asistente IA

Problema

Las respuestas largas no entraban en el panel derecho.

Corrección

Se agregó scroll propio al resultado del Asistente IA.

Ahora se pueden ver

 propuesta
 acciones propuestas
 plan de ramas
 datos faltantes
 advertencias
 JSON técnico

Archivos tocados

```text
Stylesworkflow.ai.assistant.css
WorkflowUI.aspx
```

Se actualizó versión de recursos

```text
Stylesworkflow.ai.assistant.cssv=dev002
Scriptsworkflow.ai.assistant.jsv=dev002
```

### fix7 — Resolver global de DocTipo y campos

Problema

En frases largas, aunque el usuario decía `NC`, el sistema podía seguir preguntando

```text
¿Qué tipo de documento querés cargar
No pude determinar el campo CAE porque falta confirmar el DocTipo.
No pude determinar el campo total porque falta confirmar el DocTipo.
```

Corrección

Ahora el DocTipo se busca globalmente en todo el texto completo, contemplando puntuación

```text
NC,
NC.
factura,
nota de crédito,
```

Resultado validado

```text
NC → NOTA_CREDITO_ELECTRONICA_AR
contextPrefix → notaCredito
CAE → biz.notaCredito.cae
total → biz.notaCredito.total
```

Versión interna

```text
1.4-mlnet-docfield-resolver
```
