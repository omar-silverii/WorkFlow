6. Cobertura semántica validada
A — No supera / caso contrario

Valida frase con:

Si no supera 200000 → COMPRAS
Caso contrario → DIR_GENERAL
DIR_GENERAL APTO → Logger Info
DIR_GENERAL NO APTO → Logger Warn

Esperado validado:

control.if total > 200000
TRUE/SI → DIR_GENERAL
FALSE/NO → COMPRAS
phraseSemanticConsistencyOk = true
warnings vacío
errors vacío
B — Notificaciones por rama

Valida:

Si total > 200000 → DIR_GENERAL
Si Dirección aprueba → notify COMPRAS
Si Dirección rechaza → logger Warn + notify COMPRAS
Caso contrario → logger Info

Quedó corregido que el auditor no exija logger cuando la frase pide notify.

C — IF compuesto ANY positivo

Valida:

CAE no vacío OR comprobante asociado no vacío
rulesMode = any
SI → ADM_FIN
NO → COMPRAS

Semántica completa OK.

D — IF compuesto ANY negativo

Valida:

Si no tiene CAE o falta comprobante asociado
rulesMode = any
CAE empty
comprobante empty
SI → COMPRAS
NO → ADM_FIN

Semántica completa OK.

E — IF compuesto ALL

Valida:

Tiene CAE
total > 200000
itemsCount > 0
rulesMode = all
SI → DIR_GENERAL
NO → COMPRAS
DIR_GENERAL APTO → Logger Info
DIR_GENERAL NO APTO → Logger Warn

Semántica completa OK.

F — Múltiples tareas humanas

Valida:

COMPRAS NO APTO → Logger Warn
ADM_FIN APTO → Logger Info
