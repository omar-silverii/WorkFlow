
* No toca handler.
* No toca motor.

Funcionamiento:

```text
control.retry opera sobre el nodo siguiente.
```

Ejemplo:

```text
Inicio
→ Reintentar
→ Solicitud HTTP
→ Logger
→ Fin
```

Prueba OK con HTTP `/Api/Ping.ashx`:

```text
[Retry] Reintento HTTP fix34 (reintentos=3, backoffMs=500)
[Retry] intento 1/4 -> n3 (http.request)
[HTTP] Client=intranet GET http://localhost:55557/Api/Ping.ashx => 200
[Logger] [Info] Terminó retry. Status=200
[End]
```

Prueba con URL inexistente:

Configuración:

```text
Reintentos = 2
Backoff = 500
```

Logs:

```text
[Retry] intento 1/3
[HTTP] ... /Api/NoExiste_fix34.ashx => 404
[Retry] backoff 500ms

[Retry] intento 2/3
[HTTP] ... => 404
[Retry] backoff 500ms

[Retry] intento 3/3
[HTTP] ... => 404
[Logger] [Info] Terminó retry con status=404
[End]
```

Observación UX:

* El handler interpreta `Reintentos = 2` como:

  * 1 intento inicial
  * 2 reintentos adicionales
  * total 3 intentos
* Funcionalmente está correcto.
* Posible fix futuro menor: renombrar etiqueta a:

```text
Reintentos adicionales
```

o mostrar ayuda:

```text
Total de intentos = 1 intento inicial + reintentos
```
