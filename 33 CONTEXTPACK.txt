## 15. Pendiente importante detectado y guardado

`WF_Notificacion.Leido` hoy es global por registro.

Problema:

```text
Si una notificación va dirigida a un rol,
cuando un usuario la marca como leída,
queda leída para todos los usuarios del rol.
```

Para una primera versión está aceptable, pero operativamente lo correcto es lectura por usuario.

Pendiente recomendado para próxima sesión:

```text
fix19 — lectura de notificaciones por usuario
```

Posible diseño:

```text
WF_Notificacion = cabecera/mensaje/destino general
WF_NotificacionLectura = estado de lectura por usuario
```

Tabla sugerida:

```text
WF_NotificacionLectura
- Id
- NotificacionId
- Usuario
- Leido
- FechaLeido
```

O más simple:

```text
Id
NotificacionId
Usuario
FechaLeido
```

Criterio:

* Para notificación directa a usuario: lectura afecta solo a ese usuario.
* Para notificación a rol: cada usuario del rol debe poder marcarla como leída individualmente.
* El badge debe contar no leídas para el usuario actual considerando:

  * notificaciones directas no leídas
  * notificaciones de roles del usuario que aún no tengan lectura por ese usuario.
