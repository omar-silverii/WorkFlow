13. Tema futuro importante: IF compuesto

Omar planteó que el IF actual está limitado:

campo operador valor

Ejemplo:

biz.notaCredito.total > 200000

Pero un workflow real necesita condiciones compuestas:

CAE no está vacío
Y total > 200000
Y itemsCount > 0

O:

CAE no está vacío
O comprobante asociado informado

Decisión:

No mezclar con fix39d/fix39e. Dejar como mejora funcional futura:

fix40 — IF compuesto / Condición compuesta

Alcance pensado:

varias reglas dentro del mismo IF
modo:
todas las reglas deben cumplirse = Y / AND
cualquiera debe cumplirse = O / OR
operadores:
igual
distinto
mayor
menor
mayor o igual
menor o igual
contiene
no contiene
está vacío
no está vacío
verdadero
falso
valor puede ser:
texto fijo
número
fecha
otro campo del contexto

Ejemplo deseado:

Condición: Validar NC completa

Modo:
Todas deben cumplirse

Reglas:
1. biz.notaCredito.cae no está vacío
2. biz.notaCredito.total mayor que 200000
3. biz.notaCredito.itemsCount mayor que 0

Antes de implementar hay que revisar el handler real control.if, porque puede ser necesario extender el motor/handler o generar internamente estructuras compatibles.
