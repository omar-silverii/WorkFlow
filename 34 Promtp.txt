### Problema detectado

En el constructor, para `Notificación interna`:

```text
Destino = Rol
```

muestra correctamente un combo de roles.

Pero para:

```text
Destino = Usuario
```

hoy deja escribir manualmente.

Eso puede generar errores.

### Resultado esperado

Cuando el usuario elige:

```text
Destino = Usuario
```

debe mostrarse un combo buscable/autocomplete con usuarios reales.

Debe permitir escribir una parte del usuario y filtrar.

Ejemplos:

```text
OMARD\OMARD
OMARD\USUARIO1
OMARD\USUARIO2
OMARD\USUARIO3
OMARD\USUARIO4
OMARD\USUARIO5
```

El usuario debería poder escribir:

```text
oma
usuario
OMARD
```

y seleccionar de la lista.

### Revisar si aplica también a

```text
Tarea humana asignada a Usuario
```

Pero hacerlo con cuidado y sobre código real.

### También considerar en fix24f

Mejorar frase cuando hay un paso común después de un IF.

Ejemplo actual:

```text
si se cumple → tarea COMPRAS
si no se cumple → enviar correo

```

Ese `luego notificar` es un paso común después del IF.

La frase debería dejar más claro que se ejecuta después de cualquiera de las ramas, por ejemplo:

```text
luego de cualquiera de las dos ramas notificar internamente...
```

No implementar todavía un validador funcional completo.

Solo mejorar si es simple y seguro.

fix25 — ramas por resultado de tarea humana

## Concepto clave de fix25

Separar dos tipos de decisión:

```text
IF automático:
decide el sistema con datos del contexto.

Tarea humana:
decide una persona al cerrar una tarea.
```

Ejemplo:

```text
Si total > 100000 → ADM_FIN
```

es IF automático.

Pero:

```text
Si ADM_FIN aprueba → COMPRAS
Si ADM_FIN rechaza → DIR_GENERAL
```

es rama por resultado humano.

## Resultado esperado de fix25

En el constructor, cuando se agregue una tarea humana, permitir definir resultados/salidas.

Ejemplo:

```text
Tarea humana:
Rol: ADM_FIN
Objetivo: aprobar nota de crédito

Resultados:
- Aprobado → mandar tarea a COMPRAS
- Rechazado → mandar tarea a DIR_GENERAL
- Observado → notificar a GERENCIA o volver a otro paso
```

No hace falta resolver todo de golpe. Empezar con una versión mínima profesional:

```text
Aprobado
Rechazado
```

Y permitir indicar qué paso agregar en cada resultado.
